# Task Decomposition

## Use When
- A recommendation is accepted for implementation
- A task needs to be split into smaller executable pieces
- Success criteria need to be made concrete

## Guidance
- Preserve the original recommendation as the source of truth
- Create tasks that are independently verifiable
- Keep task scope minimal and actionable
- Link each task back to its parent recommendation
