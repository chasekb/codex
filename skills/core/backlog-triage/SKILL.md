# Backlog Triage

## Use When
- New backlog items need classification
- A recommendation needs priority or scope review
- Noise must be separated from real work

## Guidance
- Keep the canonical backlog small and reviewable
- Classify items as recommendation, task, or noise
- Prefer explicit priority and dependency notes
- Promote only accepted recommendations
