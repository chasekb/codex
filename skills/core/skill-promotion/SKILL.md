# Skill Promotion

## When To Use
- a prompt or learning signal describes a reusable behavior gap
- a recurring pattern should become a local skill
- an existing skill needs to be rewritten or tightened for reuse
- a skill candidate needs a concise draft before implementation

## Workflow
1. Read the backlog candidate and learning context.
2. Decide whether the best path is create, update, or defer.
3. Draft the smallest reusable skill needed for the recurring behavior.
4. Keep runtime policy out of the skill body.
5. If the better fit is an external skill, hand off to skill maintenance instead of duplicating it.

## Output
- recommended skill name
- trigger text
- short description
- minimal instructions
- whether the candidate should become a new skill or update an existing one

## Constraints
- Keep the skill self-contained and reusable.
- Prefer behavior-shaping content over policy.
- Do not expand into runtime routing, budgets, or memory rules.
